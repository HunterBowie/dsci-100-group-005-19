Questions¶

Broad Question (picked #2)¶

Which "kinds" of players are most likely to contribute a large amount of data?

Specific Question¶

Can Age and experience predict played_hours in players.csv?

Since players who have higher played_hours will be contributing more data in the form of more sessions, we can use played_hours as a determination of data contribution. The "kind" of player that contributes more data can be characterized using their age and level of experience. the Age predictor will have to be standardized if I use k-nearest-neighbor as the model to predict played hours.

Methods and Plan¶

Method¶

Mode: Regression

Engine: K nearest neighbor

This is regression problem because we are trying to predict a quantitive variable (played_hours).

The best engine to use for this problem is K nearest neighbor because there is no consistent linear trend in the predictors for linear regression.

Limitations/Weaknesses¶

K nearest neighbor will become very inaccurate near the edges of our data when we have less points for it to be near. In this case outsized Age values will probably give inaccurate predictions (often flatlining).

Comparing the Model¶

I will compare the model using the RMSPE value which expresses the prediction error. I will use cross validation to select the K for my model with the lowest RMSPE.

Processing the Data¶

I will need to split the data into training and testing data with a higher proportion being for training. I will have 75% be training data and 25% be testing data. I will split the data before I begin building my model because I must use only the training data for my cross model training and cross validation.
